## Field guide

- **Title** — Short and recognisable. What players will scan for.
- **Premise** — One sentence that sells the fantasy. Save the detail for the rules.
- **Type** — One or more category tags. A challenge can belong to several (e.g. Occult *and* Villain) — that's expected.
- **Difficulty** — A 1–5 rating (shown as plumbobs on the site) plus a word. Keep author ratings honest; community votes can adjust it later.
- **Length** — Set expectations up front so nobody starts a 10-generation run thinking it's an afternoon.
- **Packs required vs recommended** — *Required* gates compatibility (the pack filter hides the challenge if a player lacks one). *Recommended* never blocks play. Be precise — this is what makes the whole site work.
- **Objective** — Must be measurable. "Get rich" is weak; "Reach §100,000 with no paid career" is a real win condition.
- **Fail conditions** — Tell players exactly how a run ends. If there's no fail, say so — vagueness is the thing this site exists to kill.
- **Setup** — Everything to configure *before* play, so two players start identically.
- **Hard rules** — The non-negotiables. Keep them numbered, short, and unambiguous. Aim for 3–6; if you need more, the challenge may be two challenges.
- **What you can do (Allowed)** — The permitted side of the boundary: which cheats are OK (e.g. `bb.moveobjects`), which items/actions/mods are fine. Removes the guesswork that vague rule sets leave behind.
- **What you can't do (Not allowed)** — The explicit prohibitions. Pair it with Allowed so a player never has to guess whether something is "in the spirit" of the challenge.
- **Objectives** — A *concrete, checkable* completion list: named skills, aspirations, collections, careers, and hard numbers ("Max Wellness", "Reach §100,000", "Own three horses"). This is the difference between a real challenge and a vibe. The single **Objective** field up top is the one-line win condition; **Objectives** breaks it into tickable targets; **Milestones** is the ordered progress path. They can overlap.
- **If you don't own the packs (base-game version)** — How to play without the recommended packs: base-game substitutes for each pack-gated rule, or an honest "Requires [pack] — no base-game version." Essential for a pack-aware site — never pretend a pack-only challenge is base-game.
- **Generation ladder** — Only for themed multi-generation legacies that assign a different theme to each generation. Delete it for everything else.
- **Want it harder?** — Keep *one* clean variant here. Players asked for ways to tune difficulty up without rewriting the rules — give them exactly that, not a second rulebook.
- **Milestones** — Double as the player's progress checklist. Order them as a natural path; the last one should equal the objective.
- **Scoring** — Optional, for the competitive crowd. Most narrative challenges won't need it.
- **Credit** — Always attribute the original creator if it's a known community challenge. This community cares about it, and it protects you.
- **Last tested** — A patch/date stamp. This is how players trust the rules still work, and how you spot challenges due for a review.
